# Wonder Pilates 웹사이트 목표 문서 v0.1

## 1. 프로젝트 한 줄 정의

경이로운 필라테스(Wonder Pilates)의 **홈페이지(단일 Home 페이지)**를 프론트엔드로 먼저 제작하여, 빠르게 화면을 확인·개선하고, 추후 SEO/AEO/GEO 최적화 및 CMS/백엔드 확장을 위한 기반을 마련한다.

## 2. 비즈니스 목표

1. **문의(리드) 증가**: 홈페이지 방문자가 상담 문의로 전환되도록 만든다.
2. **매출 증가**: 문의 → 방문 상담 → 등록(일반 수업/향후 지도자 과정)으로 이어지게 한다.
3. **브랜드 신뢰 형성**: "경이로운 필라테스만의 수업 철학/대표 전문성"을 한 페이지 안에서 명확히 전달한다.

## 3. 이번 스프린트 범위

### 포함

* **프론트엔드 단일 페이지(Home `/`) UI 제작**
* 모던하고 여성스러운 무드의 디자인
* 핵심 섹션 구성(가치/프로그램/대표/문의)
* 상단/하단 및 핵심 영역에 **3채널 컨택트 CTA** 배치(전화/카카오/네이버 톡톡)
* 모든 텍스트/데이터를 한 파일 SSOT(content layer)로 분리(i18n-ready)

### 제외

* 백엔드/DB/Supabase/Directus/CMS 연동
* 블로그/다중 페이지/콘텐츠 운영 시스템
* 장기 SEO 실행 플랜, 대량 콘텐츠 기획

## 4. 성공 기준

* **1차 성공(개발/디자인)**: Home 페이지가 실행 가능하고(Preview), 주요 섹션과 CTA가 의도대로 동작한다.
* **2차 성공(UX/전환)**: 홈에서 문의 버튼이 명확하고, 사용자가 '어떤 곳인지/무엇을 받는지/왜 신뢰할 수 있는지' 30초 내 이해 가능하다.

## 5. 사용자 전환 목표

* 최종 행동: **문의 클릭(우선)**
* 컨택트 채널:

  * 네이버 톡톡
  * 카카오 채널
  * 전화
* UI 원칙: CTA는 눈에 잘 띄고, 반복 노출되며, 클릭 가능한 요소로 설계한다.

## 6. 브랜드/카피 핵심 메시지

* 핵심 차별점: **웨이트·요가·필라테스의 통합 관점**과 **10년+ 현장 지도 경험**을 수업에 녹여 "현장에서 실제로 겪는 케이스"를 다룬다.
* 센터/수업 톤: 모던·여성스러움·정돈된 전문성

## 7. 대표(Founder) 신뢰 요소

* 인물: 김현경 대표이사
* 현 역할:

  * 現 경이로운필라테스 대표
  * 現 K클래식필라테스 협회 대표이사
* 자격/이력(요약): 생활체육지도자 2급, RYT300, 국제필라테스, 운동처방사 등
* 출간 교재: Pilates Anatomy 1/2, Foundation, 기구별 Advanced/Intermediate 교재
* 설립 이력: 2022 BPIA 설립, 2024 KCPA 설립 및 교재 출간

## 8. Home 페이지 IA(섹션 구성)

1. Header

   * 브랜드(한/영) + 앵커 네비(#programs #founder #contact)
   * CTA 3개(톡톡/카카오/전화)
2. Hero

   * H1: 경이로운 필라테스
   * 짧은 서브카피(수정 가능)
   * Primary CTA: 톡톡
3. Why Wonder(가치 제안)

   * 3~5 카드: 통합 관점, 10년+ 경험, 현장 적용 중심 등
4. Programs Preview(프로그램/수업 미리보기)

   * 4 카드(placeholder): 그룹/개인/체형교정/컨디셔닝(재활)
5. Founder(대표 소개)

   * 프로필 카드 + 신뢰 요소 리스트
6. Contact

   * 지도/주소는 placeholder 박스(추후 입력)
   * CTA 3개 반복
7. Footer

## 9. 시스템 설계 원칙

* **하드코딩 금지**: 연락처/주소/운영시간/카피/링크는 UI 컴포넌트에 박지 않는다.
* **SSOT content layer**: `content/home.ts` 같은 단일 파일에서 모든 텍스트/데이터를 관리한다.
* **i18n-ready**: KR/EN 필드를 구조로 마련해, 향후 영문 페이지 확장 가능.

## 10. 개발자에게 전달할 실행 요구사항

* Next.js(App Router) + Tailwind + shadcn/ui 기반
* 라우트는 `/` 하나만 생성
* 컴포넌트는 섹션 단위로 분리 가능
* 메타(title/OG/canonical)는 요구 표준에 맞춤
* 실행 명령어와 Preview 안내 포함
